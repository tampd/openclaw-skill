# Content Guidelines — Chaiko Tech Blog

> Đọc file này khi viết/review/đăng bài để đảm bảo chất lượng nhất quán.

## Định vị Blog

- **Tên**: Chaiko Tech Blog
- **Đối tượng**: Developer, IT professional, doanh nghiệp nhỏ tại Việt Nam
- **Ngôn ngữ**: Tiếng Việt (có thể mix thuật ngữ tiếng Anh khi cần)
- **Tone**: Chuyên nghiệp nhưng dễ tiếp cận, không quá academic
- **Mục tiêu SEO**: Ranking cho keyword tiếng Việt về AI, công nghệ, bảo mật

## SEO Checklist (Yoast SEO v27)

### BẮT BUỘC trước khi publish
- [ ] **Tiêu đề**: 30-60 ký tự, chứa focus keyword, hấp dẫn
- [ ] **Meta description**: 120-155 ký tự, tóm tắt giá trị bài viết, có CTA
- [ ] **Focus keyphrase** (Yoast): 1 keyword chính cho mỗi bài
- [ ] **Slug**: ngắn, chứa keyword, dùng tiếng Việt không dấu + gạch ngang
- [ ] **H1**: chỉ 1 H1 = tiêu đề bài, chứa keyword chính
- [ ] **Heading hierarchy**: H2 → H3 → H4, không skip level
- [ ] **Nội dung**: ≥ 800 từ (tối ưu 1500-2500 từ)
- [ ] **Hình ảnh**: ≥ 1 featured image, alt text chứa keyword
- [ ] **Internal links**: ≥ 2 link đến bài cùng category
- [ ] **External links**: ≥ 1 link tới nguồn uy tín

### Cấu trúc bài viết chuẩn
```
H1: Tiêu đề (chứa keyword)
├── Đoạn giới thiệu (hook reader, 2-3 câu)
├── H2: Phần 1
│   ├── Nội dung
│   └── H3: Chi tiết (nếu cần)
├── H2: Phần 2
│   └── ...
├── H2: Kết luận / Tổng kết
└── H2: FAQ (nếu có — tốt cho SEO)
```

### Format tiêu đề
- **Hướng dẫn**: "Cách [Action] + [Object] + [Năm]" hoặc "Hướng Dẫn Chi Tiết [Topic]"
- **Phân tích**: "So Sánh [A] vs [B]: [Tiêu chí] + [Năm]"
- **Nghiên cứu**: "[Topic]: [Insight] + [Năm/Tương lai]"
- **Bảo mật**: "[Mối đe dọa/CVE]: [Hậu quả] và Cách Phòng Tránh"

## Quy tắc hình ảnh

1. **Featured image**: 1200×630px (tối ưu social sharing)
2. **Alt text**: mô tả ảnh + chứa keyword, tiếng Việt
3. **File name**: slug-keyword-descriptive.jpg (e.g. `so-sanh-vps-hosting-2026.jpg`)
4. **Format**: WebP hoặc JPEG (< 200KB)
5. **Không dùng**: ảnh stock generic, ảnh watermark, ảnh blur

## Internal Linking Strategy

| Category | Link tới |
|---|---|
| Nghiên cứu AI | → Phân tích (bài liên quan) |
| Hướng dẫn | → Công nghệ lõi (prerequisite) |
| Bảo mật | → Hướng dẫn (cách fix) |
| Phân tích | → Tương lai (dự đoán liên quan) |

## Lịch đăng bài đề xuất

| Ngày | Loại bài |
|---|---|
| T2 | Nghiên cứu AI / Phân tích (dài, chuyên sâu) |
| T4 | Hướng dẫn / Tutorial (thực hành) |
| T6 | Bảo mật / Tin nhanh / Trend |

- **Tần suất**: 2-3 bài/tuần
- **Giờ đăng tối ưu**: 09:00 hoặc 20:00 (GMT+7)
